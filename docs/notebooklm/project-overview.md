# 校車管理系統後端專案總覽

## 一句話介紹

這是一套以 `ASP.NET Core 8` 建置的校車管理後端系統，針對學校情境設計了每週搭車登記、教師車上點名、管理端路線管理、跨路線調度、通知提醒與報表匯出等能力，並可直接透過 Swagger 進行角色切換與功能展示。

## 專案目標

本專案的目標不是只做出 CRUD API，而是做出一個可 Demo、可解釋、可延伸的後端作品，讓面試官可以快速理解：

- 這個系統解決了什麼實際問題
- 系統中有哪些角色與權限差異
- 後端如何處理資料模型、驗證授權與商業規則
- 專案是否具備可測試、可維護、可展示的工程結構

## 問題背景

校車管理在真實場景中通常會遇到幾類問題：

- 家長每週都要替孩子填寫搭車需求，容易重複操作
- 教師需要在車上快速點名，並在必要時找到家長聯絡方式
- 管理者需要處理臨時調度、路線異動與公告通知
- 學生、家長、教師、管理員看到的資料範圍並不相同

這個專案就是針對這些場景，把資料、流程與角色權限一起設計完整。

## 系統主要角色

### 管理員

- 管理路線與停靠站
- 指派教師到路線
- 建立單次跨路線調度
- 發送全域公告
- 產出報表
- 手動執行提醒作業

### 教師

- 查看自己被指派的路線
- 依據當天搭乘名單建立點名場次
- 更新學生點名狀態
- 完成點名 session

### 家長

- 查看孩子每週搭車登記資料
- 更新該週的上學與放學搭乘安排
- 套用上一週設定，減少重複輸入
- 查看孩子已登記與實際搭乘摘要

### 學生

- 查看自己的搭車資料與摘要
- 在前端情境中可依角色限制只讀取自己的資料

## 核心功能模組

### 1. 驗證與登入

- 支援 `account + password` 登入
- 不同角色使用不同型態帳號
- 學生用學號
- 教師與管理員用工號
- 家長用手機號碼
- 登入成功後回傳 JWT access token 與 refresh token

### 2. 每週搭車登記

- 以一週為單位管理學生的搭車安排
- 每天可分別設定上學與放學是否搭乘
- 可指定上下學不同路線
- 可直接複製上一週設定

### 3. 路線與站點管理

- 建立路線主檔
- 維護停靠站順序與地址
- 支援一般路線與 Door-to-Door 類型
- 支援教師與路線的指派關係

### 4. 車上點名

- 教師可根據當天登記資料建立點名 session
- 每一筆點名紀錄都對應到學生
- 點名資料會保留緊急聯絡電話快照
- 完成後可將整個 session 標記為完成

### 5. 通知與提醒

- 管理員可手動執行提醒作業
- 系統也有背景提醒服務
- 可查看通知寄送歷程
- 管理員可發送廣播公告

### 6. 調度與報表

- 可針對指定學生建立某一天某方向的路線覆寫
- 可匯出報表並下載 CSV 檔案

## 技術亮點

- 採用模組化單體設計，模組邊界清楚
- 使用 `ASP.NET Core Identity + JWT` 管理登入與授權
- 使用 `EF Core` 管理資料存取與 migration
- 以 Swagger 作為後端展示與文件入口
- 啟動時自動 migration 與 seed demo data
- 整合 `Serilog` 與 health checks
- 測試包含 Domain、Application 與 API Integration Tests

## Demo 時最值得強調的價值

- 這不是只有登入與清單查詢的展示專案，而是包含角色、流程與資料關係的完整系統
- 所有角色都能透過同一組 API 被實際驗證
- 文件、Swagger、seed data 與測試都有對齊
- 面試時可以很快展示「業務流程」與「工程能力」兩條線
